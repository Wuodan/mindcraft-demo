# Developer Notes

## Regenerating `minecraft-config/bukkit.yml` for another Minecraft version

`bukkit.yml` is generated by the Minecraft server startup process and can be version-sensitive.
For this repo, we only customize:

- `settings.connection-throttle: 0`

Paper docs describe `connection-throttle` as the delay between connection attempts from the same IP (milliseconds).
Default is typically `4000`; setting it to `0` disables that delay.

When changing Minecraft version, regenerate from the same server image/version combo and then re-apply that one change.

### Quick recipe

1. Start a temporary server container for the target version:

```shell
docker run -d --name mc-bukkit-bootstrap \
  -e EULA=TRUE \
  -e TYPE=PAPER \
  -e VERSION=1.21.6 \
  itzg/minecraft-server@sha256:cce41161e946a6a34ee6993b093bf3ed9a3942bd6b2fcf24d9d2797941884b4e
```

2. Wait until startup finishes (or at least until config files are generated):

```shell
docker logs -f mc-bukkit-bootstrap
```

3. Export generated `bukkit.yml`:

```shell
docker exec mc-bukkit-bootstrap cat /data/bukkit.yml > minecraft-config/bukkit.yml
```

4. Edit `minecraft-config/bukkit.yml` and set:

```yaml
settings:
  connection-throttle: 0
```

5. Clean up the temporary container:

```shell
docker rm -f mc-bukkit-bootstrap
```

### Notes

- Keep `TYPE` and `VERSION` aligned with your `docker-compose.yml`.
- If you change the pinned `itzg/minecraft-server` image digest, regenerate this file again.
